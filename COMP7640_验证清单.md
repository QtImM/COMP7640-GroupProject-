# COMP7640 课程项目验证流程

本文档把 BU Marketplace 的验证过程整理成按角色拆分的 Markdown 流程，分为顾客登录验证和商家登录验证两条主线。你可以直接按顺序操作，逐步确认功能是否符合 `COMP7640-Course Project requirements.pdf` 的要求。

## 1. 验证前准备

开始验证前，先确认这些前置条件都已完成：

1. 已安装 Python 虚拟环境并完成依赖安装。
2. 已创建本地 MySQL 数据库 `comp7640_marketplace`。
3. 已导入项目提供的基础表结构和示例数据。
4. `database.yaml` 已正确配置本地数据库连接信息。
5. 运行入口为仓库根目录下的 `run.py`。

如果任一条件不满足，先补齐环境，再继续后面的验证步骤。

## 2. 启动项目

1. 打开终端，进入仓库根目录。
2. 激活虚拟环境。
3. 启动应用：`py run.py`。
4. 打开浏览器访问首页。

预期结果：页面能正常打开，没有数据库连接错误、模板错误或导入错误。

## 3. 顾客登录验证

### 步骤 1：顾客注册

1. 打开 `/customerRegister`。
2. 填写姓名、邮箱、手机号、密码和收货地址。
3. 提交表单。

预期结果：系统提示注册成功，并跳转到顾客登录页。

### 步骤 2：顾客登录

1. 打开 `/UserLogin`。
2. 输入刚刚注册的邮箱和密码。
3. 完成登录。

预期结果：登录后进入顾客首页 `/home/<user_id>`，并显示顾客身份。

### 步骤 3：浏览商品

1. 在首页查看全部商品列表。
2. 点击某个商品进入详情页。

预期结果：页面能正常展示商品名称、价格、库存、标签、描述和供应商信息。

### 步骤 4：按供应商浏览

1. 打开 `/vendors`。
2. 选择一个供应商并进入其商品列表。
3. 或者直接访问带 `vendor_id` 的首页。

预期结果：页面只显示该供应商的商品，不会混入其他供应商的商品。

### 步骤 5：搜索商品

1. 在首页搜索框输入商品名关键词。
2. 再输入一个标签关键词。

预期结果：系统都能返回匹配结果，支持模糊搜索。

### 步骤 6：加入购物车

1. 在商品页点击加入购物车。
2. 返回订单页查看购物车内容。

预期结果：商品成功进入购物车，数量和小计正确。

### 步骤 7：调整购物车数量

1. 在订单页增加商品数量。
2. 再减少商品数量。

预期结果：数量更新正确，不会超过库存；减少到 0 时商品应从购物车移除。

### 步骤 8：提交订单

1. 打开 `/placeOrder/<user_id>`。
2. 填写收货信息和付款方式。
3. 提交下单。

预期结果：系统创建主订单，订单状态为 `placed`，并生成对应订单项和交易记录。

### 步骤 9：查看订单历史

1. 打开 `/myOrders/<user_id>`。
2. 展开订单详情。

预期结果：能看到订单主信息、订单项明细、金额和状态。

### 步骤 10：取消未发货订单

1. 找到状态仍可修改的订单。
2. 执行 `/cancelOrder/<user_id>/<order_id>`。

预期结果：订单状态变为 `cancelled`，库存恢复，交易记录同步变更。

### 步骤 11：删除未发货订单项

1. 找到一个未发货订单中的具体订单项。
2. 执行 `/removeOrderItem/<user_id>/<order_id>/<order_item_id>`。

预期结果：该订单项被移除，剩余订单金额重新计算；如果删到没有订单项，订单应转为取消状态。

## 4. 商家登录验证

### 步骤 12：商家注册

1. 打开 `/sellerRegister`。
2. 填写商家姓名、邮箱、密码、联系电话和经营地点。
3. 提交表单。

预期结果：系统提示注册成功，并跳转到商家登录页。

### 步骤 13：商家登录

1. 打开 `/SellerLogin`。
2. 输入商家邮箱和密码。
3. 完成登录。

预期结果：登录后进入商家商品管理页 `/sell/<vendor_id>`，并显示商家身份。

### 步骤 14：新增商品

1. 在商品管理页填写商品名称、价格、库存、描述和标签。
2. 提交新增商品。

预期结果：商品写入数据库，并出现在商品列表中。

### 步骤 15：编辑商品

1. 选择已有商品。
2. 修改名称、价格、库存、描述或标签。
3. 保存修改。

预期结果：修改后的内容会同步到前台展示，标签数量不能超过系统限制。

### 步骤 16：下架商品

1. 在商品管理页对某个商品执行下架操作。

预期结果：商品状态变为非激活，前台不再作为可售商品展示。

### 步骤 17：查看入站订单

1. 在商家商品管理页 `/sell/<vendor_id>` 点击“订单履约 / Fulfillment”按钮入口。
2. 跳转到 `/vendorOrders/<vendor_id>`。
3. 查看该商家的订单项列表。

预期结果：只显示当前商家相关的订单项和客户信息。

### 步骤 18：发货订单项

1. 在商家订单页对一个未发货订单项执行发货。
2. 检查主订单状态是否随剩余未发货项变化。

预期结果：顾客端可看到订单项状态变为 `shipped`，主订单状态会根据剩余未发货项自动更新为 `processing` 或 `shipped`。

## 5. 验证结果判断

如果下面这些结果都成立，就可以认为验证通过：

1. 顾客登录后可以正常完成浏览、搜索、加购、下单和订单管理。
2. 商家登录后可以正常完成商品管理和订单发货。
3. 顾客和商家的身份隔离正确，页面入口和功能权限不混淆。
4. 商品、订单、订单项、交易记录的数据流转一致。
5. 库存不会被超卖，取消与删除订单项后金额和库存会正确回滚或重算。
